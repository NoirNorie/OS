본인처럼 리눅스를 모르는 사람도 어찌되었든 하게 만드는 교수님의 과제들

./mlfq.py -Q 9,9,9,9 -i 0 -l 0,590,24:0,590,16:0,590,8 -S -c

./mlfq.py : 실행 파일

Q 9,9,9,9
- 각 큐의 양자값(Quantum / Time slice)은 학번 끝자리 + 2 ( 끝자리가 7이어서 +2 => 9)
- 큐는 4개를 사용
-> 양자값이 9인 큐 4개 설정

i 0
- 인터럽트 고정상수를 의미
- 치팅을 하게 될 경우 0으로 설정하면 계산하기 편리함


l 시작시간, 총 작업수행시간, 동작시간 : ... : ...
- ' : ' 로 작업을 구분함 
-> 3개의 작업이 필요하다 = 작업 1 : 작업 2 : 작업 3

- 시작시간 = 0
-> 문제에 모든 작업이 시간 0에서 시작하겠다고 했으므로 

- 총 작업수행 시간 : 이것으로 소수점을 조절해야 함 
-> 시행회수를 늘리면서 소수점을 맞춰 가볼 것

- 동작시간
-> 분명 치팅을 사용하라고 하실테니 어떤 작업은 동작시간이 양자값보다 작아야 할 것임
-> 양자값보다 작업 시간이 짧으면 우선순위 큐에서 하강하지 않으므로 계속 수행될 것임

** 소수점 첫째자리로 반올림
= 소수점 둘째자리에서 반올림해서 97.0%가 나와야 한다고 교수님이 답변하심
= 즉 소수점 둘째자리가 5 미만이어야 함
= 한국어는 끝까지 들어봐야 하는 것 같음

S : 치팅을 사용하겠다.
- Rule 4-b의 치팅과 관련된 동작을 적용하는 명령어

c : 계산과정을 보겠다
- 모든 과정을 출력함
